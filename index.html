🤖 Greedy Route

Always picks nearest unvisited employee from current position. Fast, near-optimal.

🧩 DP Route

Tries ALL possible subsets of employees per cab using bitmask DP. Finds true optimal for each cab.

🤖 How Greedy Works — Steps

// Greedy pseudocode
for each cab:
cur = office (0,0)
for each seat:
next = MIN dist(cur, unvisited)
assign next, cur = next
dist += return_to_office

🧩 How DP Works — Steps

// DP (Held-Karp style) pseudocode
for each cab:
pool = unassigned employees
for each subset S of pool (2ⁿ):
for each last node v in S:
dp[S][v] = min over all u
dp[S\v][u] + dist(u,v)
pick best subset of size ≤ seats

❓ Why Greedy Algorithm?

The greedy algorithm was chosen because it provides efficient route optimization with relatively low computational complexity. While dynamic programming can produce an optimal solution, its time complexity O(n² × 2ⁿ) becomes impractical as the number of employees increases. The greedy approach offers a near-optimal solution with a manageable time complexity of O(n²), making it suitable for real-world fleet routing problems.

✅ Conclusion

This project demonstrates that the Greedy Algorithm is the right choice for the Smart Fleet Management System. Dynamic Programming guarantees optimality but its exponential time complexity O(n² × 2ⁿ) makes it computationally infeasible for real-world fleet sizes (n ≥ 20).
The Greedy approach with O(n²) complexity consistently reduces total travel distance by 15–50% compared to random assignment, while running in near-instant time regardless of fleet size.
Future improvements could include Genetic Algorithms or Simulated Annealing which bridge the gap between greedy speed and DP optimality — but for this problem, Greedy strikes the ideal balance of efficiency, simplicity, and real-world practicality.
